# 全站番剧封面 A 方案卡片统一设计

## 目标

将截图中的番剧封面统一为已经确认的 A 方案“柔光玻璃杂志卡”，覆盖：

- 首页“今日放送”的大幅主卡与四张横向卡。
- 资料库收藏页的前三扇形精选舞台与其余封面网格。
- 发现页片库的封面结果网格。
- 已完成的“本季选片”海报轨道继续作为视觉与交互基准。

现有页面布局、数据顺序、打开逻辑、删除逻辑和导航行为保持不变。

## 架构

从现有 `PosterRail` 的 A 方案实现中提炼共享、纯展示的 `ArtworkCardSurface`，放在 `lib/widgets/artwork_components.dart`。页面负责封面、标题、状态、评分等内容布局；共享外壳负责卡片材质和交互。

共享外壳接收：

- 唯一标识和完整语义标签。
- 打开回调。
- 根据当前 active 状态和动画开关构建内容的 builder。
- 可选的前景操作层，用于资料库删除按钮等独立操作。
- 圆角和上浮距离，默认使用 16px 与 6px。

共享外壳不依赖 MobX、Store、Service、路由或领域模型。`PosterRailItem`、`ArchiveEntry` 与页面使用的 Map 数据均不进入共享组件。

新增共享 `ArtworkCardBadge`，统一编号、评分等胶囊的圆角、半透明纸张、描边和阴影。评分胶囊可使用深色变体，但尺寸和层次保持一致。

## A 方案视觉规则

- 卡片使用 16px 圆角。
- 日间主题使用白色到纸张色的柔光渐变；夜间主题使用深色纸张到卡片色渐变。
- 默认使用低对比白色描边；悬停、焦点或按压状态切换为天空蓝描边。
- 使用一层中性投影和一层天空蓝环境光，不加入实时模糊、Shader 或循环动画。
- 封面与信息区保持清晰分层；信息区继续使用现有标题、状态、类型和评分内容。
- 编号、评分等角标改为 9px 左右圆角的玻璃胶囊。
- 标题与元信息继续单行省略；不因超长标题改变卡片高度。

## 交互与无障碍

- 鼠标悬停或键盘焦点时，整张卡片上浮 6px，封面缩放到 1.025。
- 使用稳定的 hover 命中包络，保证上浮后的顶部仍可点击，底部空白区域不显示误导性的点击光标。
- 主操作使用 `InkWell`，支持鼠标点击、Enter 和 Space。
- 每张卡片只暴露一个完整的打开按钮语义；视觉标题、角标和封面不重复进入语义树。
- 资料库删除按钮放在共享主操作之上，拥有独立语义与命中区域；点击删除不得同时打开番剧。
- 当 `MediaQuery.disableAnimations` 为真时，禁用位移和封面缩放，保留静态描边与焦点环。
- 阴影绘制区域不得被 Grid、ListView 或精选舞台硬裁切。

## 页面适配

### 今日放送

- 保留桌面端一张大幅主卡加四张横卡的布局。
- 大幅主卡继续使用全封面、底部渐变和白色标题，但增加 A 方案玻璃外壳与封面缩放。
- 横卡继续使用左侧封面、右侧状态/标题/评分结构；封面与信息区加入圆角层次。
- 窄屏横向列表保持现有尺寸、滚动和打开行为。

### 资料库收藏

- 前三项继续保留扇形舞台、重叠关系和轻微旋转，卡片自身改用 A 方案外壳。
- 其余项目继续显示在网格中，顺序和“不重复前三项”的语义不变。
- 图片失败、少于三项、空元信息和删除后自动补位继续可用。
- 删除按钮置于主卡 Ink 层之上，避免事件串联。

### 发现页片库

- 保留现有响应式封面网格和数据加载行为。
- 编号与评分统一为玻璃胶囊。
- 卡片外壳、信息底板、悬停、焦点和键盘行为统一到共享 A 方案。
- 加载更多、筛选、空状态和错误状态不修改。

### 本季选片

- 将现有 `PosterRail` 接入共享外壳，保持已验证的滚轮、拖动、键盘、语义、减少动态效果和阴影安全区行为。
- 迁移后视觉参数与行为不得回退。

## 响应式与性能

- 共享外壳只使用隐式动画，单次 160–180ms；不创建持续 Ticker。
- 封面缩放与整卡位移只在交互状态变化时执行。
- 960、1280、1600 宽度下，今日放送、资料库和发现页均不得溢出。
- Grid/ListView 为阴影和上浮预留安全区，首尾卡片描边不得贴边裁切。
- 不新增第三方依赖或外部图片资源。

## 验证

- 共享组件测试：视觉结构、唯一语义、Enter/Space、鼠标边缘点击、前景操作隔离、深浅主题与减少动态效果。
- 首页测试：Hero 与横卡使用共享 A 外壳，窄屏列表和 960/1280/1600 布局不溢出。
- 资料库测试：扇形舞台与网格均使用共享 A 外壳，删除不触发打开，少于三项和图片失败可用。
- 发现页测试：编号与评分胶囊、网格卡片、长标题和加载更多不回退。
- 保留并运行现有 `PosterRail` 14 项交互回归。
- 完整执行 `flutter test`、`flutter analyze` 和 Windows Release 构建。

## 非目标

- 不把所有页面改成相同宽高的固定海报。
- 不改变今日放送 Hero、资料库扇形舞台或发现页网格的排版结构。
- 不修改 Store、Service、网络接口、本地数据、排序和路由。
- 不新增播放、收藏或更多菜单操作。
