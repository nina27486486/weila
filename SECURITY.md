# 安全策略

## 支持范围

当前仅对最新的 `0.4.x` 开发版本提供安全修复。更早版本可能缺少必要的网络、文件和
凭据保护。

## 私下报告

请使用 GitHub 仓库的 **Security → Report a vulnerability** 创建私密 Security
Advisory。不要在公开 Issue 中发布可直接利用的步骤、私人视频源、Cookie、API 密钥
或包含个人目录的完整日志。

报告建议包含：

- 受影响版本与 Windows 版本
- 问题类型和影响范围
- 最小复现步骤
- 可证明问题的日志或截图
- 你建议的缓解方式

维护者会尽快确认报告并评估修复，但这是个人维护的开源项目，暂不承诺固定响应时间。

## 安全边界

以下问题值得私下报告：

- 插件输入导致任意文件读写、命令执行或目录穿越
- 下载任务写出用户选择目录之外
- 日志、设置或界面泄露凭据
- 可绕过来源校验执行不受信任代码
- 构建或更新流程存在供应链风险

第三方数据源失效、内容下架、地区限制和普通播放错误通常不属于安全漏洞，可使用
Bug 模板公开反馈。
