# 薇拉 Weila：天空日记 × 动漫潮流杂志

> 本文档用于指导 Stitch 生成薇拉动漫播放器的桌面端设计稿。  
> 目标不是制作一个“动漫风 Dashboard”，而是设计一款用户愿意每天打开的看番软件。

## 1. Stitch 总指令

为 Windows 桌面端动漫播放器“薇拉 Weila”设计一套完整、连贯、可落地的产品界面。

- 画布基准：`1600 × 1000`，同时考虑 `1280 × 800` 与 `1920 × 1080`。
- 默认主题：明亮天空主题；另提供“夜空特刊”深色主题。
- 所有用户可见文本使用简体中文。
- 产品气质：清爽、梦幻、年轻、精致、有编辑品位，但不能幼稚或过度二次元。
- 视觉概念：`天空日记 60% + 动漫潮流杂志 40%`。
- 核心体验：作品图片是主角，界面像一本可以播放的动画分镜册。
- 设计必须像真实桌面软件，不要生成营销落地页、后台管理系统或手机 App。
- 使用真实比例的动漫封面与宽幅剧照占位；如需生成占位图，使用原创的天空、海岸、列车、校园、黄昏等动画氛围，不模仿具体画师或作品。
- 不要把文字画进图片，标题、评分、按钮都必须是独立 UI 图层。

## 2. 设计目标

薇拉的界面应让用户感受到：

1. 打开软件像翻开今天的动画日记，而不是进入数据后台。
2. 每一部作品像杂志中的一篇专题，有明确的主次、构图与节奏。
3. 首页可以快速继续观看，也能自然发现新番。
4. 常用功能一步可达，低频功能不抢占首屏。
5. 悬停、切换、播放等互动有“分镜变化感”，但不喧宾夺主。
6. 轻主题清透而不苍白，深主题安静而不沉闷。

## 3. 明确禁止的模板化表达

以下模式会产生明显的 AI 感，禁止使用：

- 不使用固定左侧 Dashboard 导航栏。
- 不使用固定右侧信息栏。
- 不使用渐变光球、漂浮气泡、满屏玻璃拟态。
- 不把每一个区块都装进相同的大圆角卡片。
- 不使用整卡放大、夸张阴影或发光边框作为默认悬停。
- 不堆叠胶囊按钮和标签；文本标签优先使用下划线、细分隔线和留白。
- 不使用无意义的英文装饰词，如 `FEATURED`、`TRENDING NOW`、`VIRA SELECT`。
- 不使用 Emoji 充当图标。
- 不使用紫色霓虹、赛博朋克或游戏启动器风格。
- 不使用完全对称、等宽等高、像组件展示墙一样的排版。
- 不使用大面积粉色作为主操作色；樱花粉只负责情绪与章节标记。
- 不依赖模糊滤镜制造质感；内容图片、字体层级和构图才是主要质感来源。

## 4. 信息架构

### 4.1 顶层导航

全局只保留五个稳定入口：

| 顶层入口 | 内容 |
|---|---|
| 首页 | 继续观看、今日放送、本季新番、热门推荐 |
| 发现 | 番剧、剧场版、分类浏览、排行榜 |
| 追番 | 追番列表、更新动态、追番日历 |
| 资料库 | 收藏、观看历史、稍后再看 |
| 下载 | 下载中、已完成、缓存管理 |

其他入口：

- 搜索：作为全局命令入口，位于顶部导航右侧，支持 `Ctrl + K`。
- 设置：放入头像菜单，不再占据一级导航位置。
- 通知：仅在存在真实更新时显示，不保留无内容的占位图标。
- 播放器快捷键：位于播放器帮助面板，不进入全局导航。

### 4.2 现有功能映射

设计只重组入口，不改变业务能力，也不要求修改现有服务层、模型或路由表。

| 新入口 | 现有页面或路由 |
|---|---|
| 首页 | `/` |
| 发现 / 番剧与剧场版 | `/anime-list` |
| 发现 / 分类 | `/category` |
| 发现 / 排行 | `/ranking` |
| 追番 | `/track` |
| 追番 / 日历 | `/calendar` |
| 资料库 / 收藏 | `/collect` |
| 资料库 / 历史 | `/history` |
| 下载 | `/download` |
| 设置 | `/settings` |

### 4.3 导航原则

- 顶部导航在普通页面持续可见，高度 `72px`。
- 播放器进入沉浸模式后隐藏全局导航。
- 返回按钮执行真实返回，不把用户强制送回首页。
- 页面内部分类使用文字索引与细线指示，不使用一排厚重胶囊。
- 当前栏目同时通过字重、天蓝色短线和语义标签标识，不能只依赖颜色。

## 5. 全局页面骨架

### 5.1 顶部杂志式刊头

使用横向刊头取代侧边栏：

- 左侧：薇拉图形标识 + “薇拉”文字，宽度约 `168px`。
- 中间：五个顶层导航，间距 `32px`。
- 右侧：搜索入口、主题切换、头像。
- 页面滚动时刊头可轻微收紧至 `60px`，背景从透明变为高不透明纸白色。
- 激活项下方出现 `2px` 天空蓝短线，像杂志章节标记，从左向右绘制。
- 搜索入口默认是“搜索番剧、电影、人物…”文字与放大镜组合，不做巨型输入框。

### 5.2 内容网格

- 最大内容宽度：`1488px`。
- `1600px` 画布左右边距：`56px`。
- 使用 12 栏网格，栏间距 `24px`。
- 主要内容允许跨栏、错位和局部越界，但文字基线必须整齐。
- 页面纵向章节间距：`72px`；紧密内容组间距：`32px`。
- 不给整页内容套一个大圆角容器。

### 5.3 页面标题

标题由三部分构成：

- 小型日期或章节号，如“六月刊 · 第 26 周”。
- 中文主标题，如“今天想看点什么？”
- 一条长度不超过 `96px` 的樱花粉细线或手写式短注。

章节装饰必须有语义，不能随机散落。

深色主题不是纯黑换肤；它应像入夜后的蓝色天空。纯黑 `#05070A` 仅用于播放器舞台。

### 6.2 透明与光感

- 透明只用于剧照上的文字遮罩、顶部刊头滚动状态和浮层。
- 明亮主题浮层使用 `paper` 约 `92%` 不透明度。
- 只有命令搜索、弹幕设置和播放器抽屉允许 `12–18px` 背景模糊。
- 天空感来自大面积明亮留白、宽幅图像和冷暖光线，不来自装饰性渐变球。
- 可在页面底层使用极淡的蓝到白垂直过渡，但对比度必须低于 `4%`。

### 6.3 字体

- 中文展示标题：`Noto Serif SC SemiBold`，用于 Hero 标题、栏目封面与短句。
- 中文界面与正文：`Noto Sans SC`，用于导航、按钮、信息与列表。
- 数字排名：`Noto Sans SC Bold`，使用等宽数字特性。
- 不使用手写字体承载正文；手写感只通过短线、圈注和少量倾斜注释实现。
- 不用大量英文大写制造“杂志感”。

建议字号：

| 层级 | 字号 / 行高 |
|---|---|
| Hero 标题 | `52 / 62` |
| 页面标题 | `36 / 46` |
| 章节标题 | `26 / 34` |
| 卡片标题 | `17 / 24` |
| 正文 | `15 / 25` |
| UI 标签 | `13 / 18` |
| 注释 | `12 / 18` |

### 6.4 圆角、描边与阴影

- 图片分镜：`8px`。
- 按钮、输入框：`6px`。
- 浮层与抽屉：`12px`。
- 不使用超过 `16px` 的大圆角。
- 普通内容卡片默认无阴影，依靠留白、图片裁切与 `1px` 描边分层。
- 只有浮层使用阴影：`0 16 48 rgba(35, 73, 104, 0.14)`。
- 悬停不改变布局尺寸，不让整张卡片跳起。

### 6.5 图标

- 使用统一的 Material Symbols Rounded 或 Flutter Material 线性图标。
- 常规尺寸 `20px`，播放器主控件 `24px`。
- 常规线宽视觉约 `1.75px`。
- 激活状态可从线性切换为实心，但不能混用多套风格。
- 所有仅图标按钮必须提供中文 Tooltip。

## 7. 核心组件语法

### 7.1 分镜卡 `StoryFrameCard`

不是传统的“圆角矩形 + 阴影 + 所有信息塞进去”。

- 上方为 16:9、3:2 或 2:3 图片。
- 下方标题与元信息直接落在页面背景上。
- 左上角可有“镜头 04”或更新时间，使用小号中文。
- 悬停时图片内部轻微放大至 `1.025`，外框不缩放。
- 底部天蓝细线从 `24px` 延展到图片宽度。
- 快速播放按钮在图片右下角淡入，位置固定，不推动文字。
- 有可用预览时，悬停 `450ms` 后交叉淡入第二关键帧；无预览时保持封面，不伪造视频。

### 7.2 主动作按钮

- 主按钮为天空蓝实色、白色文字、高度 `42px`。
- 次按钮为透明背景、`1px` 天空蓝描边。
- 文本按钮使用文字 + 箭头，不套胶囊。
- 悬停通过颜色加深、下划线绘制或图标位移 `2px` 反馈，不做弹跳。

### 7.3 文本索引标签

用于“全部 / 番剧 / 剧场版 / 排行”等页面内导航：

- 高度 `40px`，背景透明。
- 选中项字重提高，底部 `2px` 天空蓝线。
- 悬停项出现淡蓝底线。
- 相邻组使用 `24px` 间距或细竖线，不使用大量圆角 Chip。

### 7.4 评分

- 使用大号数字 + 一条 0–10 的细刻度线。
- 高分用天空蓝或暖金色小点强调，不使用廉价的巨大星星徽章。
- 评分来源以小字注明，如“Bangumi 8.7 / 2.4 万人”。
- 排行变化同时显示箭头与数值，不能只靠红绿颜色。

### 7.5 空状态与骨架

- 空状态使用简洁线稿：一张未放入画面的分镜框，搭配一句中文说明和明确操作。
- 不使用巨大灰色图片错误图标。
- 骨架屏必须保持最终布局比例，使用缓慢的横向柔光，不使用旋转加载圈作为页面主状态。
- 图片加载失败时保留标题与操作，并提供“重新加载”。

## 8. 页面设计

### 8.1 首页：今天的动画日记

首页是最具品牌感的页面，但首先服务于“继续看”和“发现内容”。

#### 首屏

- 顶部刊头下方先显示一条轻量日记信息：“6 月 28 日 · 周日 / 今晚有 4 部追番更新”。
- Hero 使用 12 栏非对称构图：
  - 左侧 8 栏为宽幅场景剧照。
  - 右侧 4 栏为纸白文字页，包含类型、标题、更新状态、简介和“继续播放”。
  - 图片与文字页可有 `24px` 的轻微错位，形成翻页感。
- Hero 下方不是圆点轮播，而是 `01 02 03 04` 细数字索引 + 一条进度线。
- 自动轮播周期 `7s`；用户悬停、键盘聚焦或操作后暂停。

#### 继续观看

- 标题：“接着上次的故事”。
- 使用横向 3:2 分镜条，展示播放进度、集数和剩余时间。
- 进度线直接贴在图片底部，悬停显示“继续播放”。
- 优先放在首屏下方，避免用户滚动很远才能继续观看。

#### 今日放送

- 不放在固定右侧栏。
- 使用横跨页面的时间轴，从 `18:00` 到 `24:00`。
- 当前时间使用天空蓝竖线；已播出降低透明度；即将播出显示更新时间。
- 无更新时显示一句安静的空状态，不占据大面积空间。

#### 本季新番

- 使用编辑式分镜布局，不使用机械等宽卡片墙：
  - 一张 6 栏主推荐。
  - 两张 3 栏竖版封面。
  - 下一行四张等宽小分镜。
- 排版要平衡，但不完全对称。
- 作品图片之间用留白分隔，不给整个区域套背景卡。

#### 热门推荐

- 使用“本周读者页”式榜单。
- 左侧为一张主视觉，右侧为 `01–05` 排名文字列表。
- 列表悬停时左侧主视觉交叉淡入对应作品，形成主动探索感。

### 8.2 发现：动画杂志目录

- 页头：“发现 / 六月动画目录”。
- 第一行使用文字索引：“全部、番剧、剧场版、分类、排行”。
- 主体采用可控的编辑网格，不做无限随机瀑布流。
- 筛选器收纳为一条可展开的“筛选目录”，默认只显示当前条件摘要。
- 类型、年份、状态、评分选择后在索引下方形成可删除的文字条件，不使用大量彩色 Chip。
- 排行榜作为发现页的子栏目：
  - 前三名用大号 `01 / 02 / 03` 与宽幅剧照表现，不做领奖台。
  - 第 4 名以后使用清晰的杂志目录列表。
  - 排名变化显示“↑2”“↓1”“—”，并附“较上周”说明。

### 8.3 追番与日历：本周放送手帐

- 页头：“我的追番 / 第 26 周”。
- 顶部先显示“待看 5 集、今日更新 2 部、下次更新 19:30”。
- 周一至周日使用横向文字标签，今天用樱花粉圈注和天空蓝下划线双重标识。
- 主体为按时间排序的放送清单，每行包含时间、封面小图、标题、集数、状态和播放按钮。
- 左侧可有细时间轴，但不使用传统月历大方格。
- “全部追番 / 本周更新 / 已完结”作为页内文字索引。
- 卡片上的取消追番属于次级操作，只在悬停或更多菜单中出现。

### 8.4 资料库：个人动画书架

- 统一收藏、观看历史和稍后再看，减少重复入口。
- 页头显示个人统计：“收藏 42 / 看完 18 / 本月观看 26 小时”。
- 子栏目使用“收藏、观看历史、稍后再看”文字索引。
- 收藏默认使用封面书架视图；历史默认使用带进度的时间线视图。
- 支持网格 / 列表切换，但切换按钮仅使用两个清晰图标。
- 批量管理进入独立状态后再显示复选框，日常浏览时保持干净。

### 8.5 详情页：一部作品的专题页

- 从卡片进入时，封面或剧照使用 Hero 过渡到详情页主视觉。
- 首屏使用宽幅场景图作为背景，前景由 2:3 封面、标题和元信息组成。
- 背景图只做局部暗化或纸白渐隐，不使用厚重黑色蒙层。
- 主操作顺序：“立即播放”“继续第 11 集”“加入追番”“收藏”。
- 评分使用大数字和 0–10 刻度线，显示多个来源但保持一眼可读。
- 简介默认显示 4–6 行，可展开。
- 集数区域支持网格 / 列表切换：
  - 网格适合快速选集。
  - 列表显示标题、更新时间、播放进度与来源。
- 角色板块使用角色头像、角色名、声优名和一句简短定位。
- 相似推荐采用横向分镜，不重复首页的标准封面墙。

### 8.6 播放器：夜间放映室

播放器是唯一以纯黑为主的页面，保持视频画面绝对优先。

- 进入播放器后隐藏全局刊头。
- 顶部仅保留返回、作品名、当前集数和窗口操作。
- 控制条默认隐藏，鼠标移动后渐入；停止操作 `2.5s` 后渐出。
- 进度条位于控制条上方，已播放使用天空蓝，缓存使用半透明白。
- 左侧控制：播放 / 暂停、下一集、音量、时间。
- 右侧控制：弹幕、清晰度、倍速、选集、快捷键、全屏。
- 选集不再永久占用右侧 `220px`；点击后从右侧滑出抽屉，按 `Esc` 关闭。
- 弹幕密度、透明度和字号在紧凑浮层中调整，并实时预览。
- 下一集提示在结束前 `20s` 出现于右下角，包含倒计时、取消和立即播放。
- 视频错误提示必须覆盖在画面之上，给出“重试、切换线路、复制诊断信息”。
- 快捷键面板使用两栏清单，按键帽样式克制，不做游戏 UI。

### 8.7 搜索：全局命令中心

- 点击搜索或按 `Ctrl + K` 打开宽约 `720px` 的居中浮层。
- 输入后在当前浮层显示最近搜索、联想结果和最多 5 个快速结果。
- 不允许在用户输入停顿后自动跳转页面。
- 用户按 `Enter` 或点击“查看全部结果”后进入完整搜索页。
- 完整结果页按“最相关、番剧、剧场版”分组，并保留筛选条件。
- 加载使用与结果卡比例一致的骨架屏。
- 无结果时提供纠错建议、清除筛选和返回热门推荐。
- 支持上下方向键选择、`Enter` 打开、`Esc` 关闭。

### 8.8 下载

- 页头：“离线观看”。
- 顶部仅展示存储空间、下载速度和全局暂停。
- 子栏目：“下载中、已完成、已暂停”。
- 下载项为横向列表，突出作品、集数、进度、速度和剩余时间。
- 删除与清空缓存是危险操作，使用红色文字并二次确认。
- 不把插件设置或技术日志混入下载页。

### 8.9 设置：编辑部后台

- 从头像菜单进入，仍保留全局刊头。
- 页面使用窄内容栏与左侧章节目录，而不是多层大卡片。
- 章节：“外观、播放、弹幕、下载、数据源、缓存、关于”。
- 主题预览使用两张真实微缩界面：“晴空日记”和“夜空特刊”，不能只是色块。
- 缓存管理显示占用构成、目录和清理结果。
- 数据源状态用“可用 / 响应慢 / 不可用”文字和图标共同表达。
- 高风险操作放在页面末尾，并与普通设置保持至少 `48px` 间隔。

## 9. 互动与动效

### 9.1 时间规范

| 类型 | 时长 | 用法 |
|---|---|---|
| 即时反馈 | `90ms` | 按下、图标状态 |
| 快速反馈 | `160ms` | 悬停、下划线、颜色 |
| 标准切换 | `240ms` | 标签、浮层、列表状态 |
| 场景过渡 | `420ms` | Hero、主视觉交叉淡入 |

- 进入使用 `easeOutCubic`，退出使用 `easeInCubic`。
- 不使用线性动画表现 UI 位移。
- 单个视图同时只允许 1–2 个主要动画。

### 9.2 页面进入

- 页面标题先淡入并上移 `12px`。
- 主视觉延迟 `60ms` 淡入。
- 首组三个内容项最多以 `40ms` 间隔出现，之后内容不再逐项炫技。
- 返回时保持滚动位置和筛选状态。

### 9.3 悬停

- 所有交互元素使用 `SystemMouseCursors.click`。
- 卡片悬停只改变图片内部、章节线、文字颜色或操作显隐。
- 不通过整卡缩放和高亮蓝边制造反馈。
- 悬停出现的操作不能是完成任务的唯一方式，键盘聚焦和点击同样可达。

### 9.4 焦点与键盘

- 焦点环：`2px sky`，外偏移 `2px`。
- Tab 顺序必须与视觉阅读顺序一致。
- `Ctrl + K`：搜索。
- `Esc`：关闭搜索、抽屉、弹幕设置和快捷键面板。
- 播放器继续支持空格、方向键、音量和全屏快捷键。
- 减少动效模式下取消位移、视差和逐项错峰，仅保留淡入与颜色变化。

## 11. 可访问性与状态完整性

- 正文与背景对比度至少 `4.5:1`。
- 大号标题与背景对比度至少 `3:1`。
- 颜色不是状态的唯一表达方式。
- 所有图片有替代文本语义。
- 所有图标按钮有中文 Tooltip。
- 所有页面都要设计：加载、空、错误、离线、无权限、图片失败状态。
- 浮层不能形成键盘焦点陷阱。
- 触发危险操作前必须确认，成功后提供可理解的结果反馈。
